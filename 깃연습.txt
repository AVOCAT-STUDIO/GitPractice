깃 연습



1. pc에 git 설치가 되어 있어야 한다
확인법 cmd 창에 git을 쳐보면 에러가 뜨면 설치가 안된거임

2. 형상관리 툴 (버전관리)
  깃: 버전 관리 툴로써 개발자는 자신만의 코드 작성시
       브랜치 단위를 파생하여 작업을 진행하며
       추후 버전으로 사용할지 버릴지 결정할수 있따 
   깃 허브: 깃을 클라우드화 시켜놓은 웹서버인다. 
             단 전 세계 모든 개발자들간에게 소스가 공유될수 있음으로
             일종의 커뮤니티로 발전했다 
3. git branch,   git switch,    git merge 등 명령어는 
커멘드 창에서 git 을 치면 설명이 뜬다. 


4. 깃 을 사용하여 작업하는방법

    a. git init     ---> 깃 디렉토리 안에는 깃 현재 또는 전체 깃에

5. 깃 허브 에서 클라우드 프로젝트


a. 현재 작업중인 프로젝트의 구성 파일들을 확정짓자
    즉. 버전을 만들자 한마디로 개발하려는 코드가 완성된 시점
b. 깃 허브 주소를 환경설정 파일에 등록한다. 
    현재 개발중인 프로젝트와 일대일로 대응하는 클라우드의 프로젝트를 의미한다.
